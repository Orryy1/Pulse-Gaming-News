# Experiment Promotion Rules

## Default

Experiments do not become production because they are interesting. They become production only when they reduce risk or improve visible output with evidence.

## Render Experiments

Promote only if:

- Canonical side-by-side comparison is materially better.
- Gauntlet passes.
- Forensic QA passes.
- Subtitle QA passes.
- Audio recurrence QA passes.
- Thumbnail safety passes.
- Source diversity is not materially worse than canonical.
- Contact sheet shows no obvious visual downgrade.

Do not promote if:

- It feels like a tech demo.
- It adds clutter.
- It lowers source diversity.
- It creates repeated SFX.
- It relies on thin media inventory.
- It needs subjective excuses to beat canonical.

## Platform Experiments

Promote only if:

- Tests pass.
- Failure payloads are preserved.
- Duplicate-post risk is explicitly handled.
- No browser-cookie or unofficial posting path is involved.
- Operator can see success, failure and skipped states.

## Analytics And Comments

Recommendations are allowed. Automatic scoring changes, automatic replies, likes, hearts and moderation are not allowed until a separate safety review.

## TikTok

Official API work should not consume repeated cycles while the blocker is app audit/public posting. Use dispatch packs until a safe approved route exists.
