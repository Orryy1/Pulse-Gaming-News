# Pulse Monetisation Roadmap

## Current Stage

Pulse is pre-monetisation but showing early traction. Current business context includes recent YouTube growth, comments and subscribers starting to move. The system should optimise retention and repeatable output before chasing revenue integrations.

## Revenue Milestones

## 0-30 Days

- Stabilise YouTube cadence.
- Protect canonical render quality.
- Use thumbnail safety to avoid unprofessional random-face frames.
- Track YPP progress locally.
- Add affiliate hygiene checks to product links.
- Draft sponsor/media-kit basics.

## 30-60 Days

- Connect read-only analytics snapshots.
- Build the first real Monthly Release Radar using verified sources.
- Start blog/newsletter repurposing for high-confidence stories.
- Test affiliate search links only where genuinely relevant.

## 60-90 Days

- Build a repeatable long-form format.
- Track watch-hour contribution by format.
- Package sponsor inventory around recurring series.
- Review TikTok Creator Rewards eligibility after dispatch cadence is working.

## Dashboard Requirements

Track:

- Subscriber progress
- Shorts view progress
- Public watch-hour progress
- Long-form watch time
- Affiliate clicks when available
- Estimated revenue range
- Format-level contribution

## Blockers

- Small current audience.
- Retention still low on known examples.
- TikTok public posting blocked.
- Long-form source verification is manual.

## Recommendation

Focus first on viewer retention, premium story selection and recurring formats. Revenue infrastructure should support those goals rather than distract from them.
